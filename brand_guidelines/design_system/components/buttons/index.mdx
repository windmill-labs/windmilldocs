# Buttons

Windmill uses **only 3 button types**. Each type has a specific purpose and should not be mixed arbitrarily.

## The 3 Button Types

<ButtonShowcase />

## Button Variants

Each button type comes in 3 variants:

### Text Button

- **Usage**: Standard actions with clear labels
- **Format**: Icon + text label
- **Example**: "Save changes", "Create workflow"

### Icon + Text Button

- **Usage**: When icon helps clarify the action
- **Format**: Icon + text label side by side
- **Example**: Play button + "Run script"

### Icon Only Button

- **Usage**: When space is limited (toolbars, compact interfaces)
- **Format**: Icon only with tooltip
- **Requirements**: Must have clear context and tooltip

## Usage Rules

### ✅ Do

- Use **Accent** buttons for the single most important action per view
- Use **Default** buttons for most standard actions
- Use **Subtle** buttons only in clear button contexts (toolbars, button groups)
- Provide tooltips for icon-only buttons
- Test button clarity across all 4 states

### ❌ Don't

- Use multiple Accent buttons in the same view
- Use Subtle buttons where context is unclear
- Create custom button styles outside these 3 types
- Use icon-only buttons without sufficient context
- Mix different button systems in the same interface

**Remember**: Always check these guidelines before implementing buttons to ensure you're using the right type for your use case.
