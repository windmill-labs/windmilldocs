# Buttons

Windmill uses **4 button types**. Each type has a specific purpose and should not be mixed arbitrarily.

## The 4 Button Types

<ButtonShowcase />

## Button Variants

Each button type comes in 3 variants:

### Text Button

- **Usage**: Standard actions with clear labels
- **Format**: Icon + text label
- **Example**: "Save changes", "Create workflow"

### Icon + Text Button

- **Usage**: When icon helps clarify the action
- **Format**: Icon + text label side by side
- **Example**: Play button + "Run script"

### Icon Only Button

- **Usage**: When space is limited (toolbars, compact interfaces)
- **Format**: Icon only with tooltip
- **Requirements**: Must have clear context and tooltip

## Button Hierarchy

### When to Use Each Type

**Accent Secondary** (Highest Priority)
- Main conversion CTAs on landing pages
- "Sign up", "Get started", "Download" buttons on marketing pages
- Primary conversion actions that drive business goals

**Accent** (High Priority)
- Most important action in application views
- Primary actions like "Save", "Submit", "Create"
- Only one per view/screen

**Default** (Standard Priority)
- Secondary actions and most UI interactions
- "Cancel", "Edit", "Delete", "View details"
- Your go-to button for standard functionality

**Subtle** (Low Priority)
- Tertiary actions in dense interfaces
- Toolbar buttons and action menus
- Use only when button context is clear

## Usage Rules

### ✅ Do

- Use **Accent** buttons for the single most important action per view
- Use **Accent Secondary** buttons exclusively for main Call-to-Action (CTA) on landing pages and marketing materials
- Use **Default** buttons for most standard actions
- Use **Subtle** buttons only in clear button contexts (toolbars, button groups)
- Provide tooltips for icon-only buttons
- Test button clarity across all 4 states

### ❌ Don't

- Use multiple Accent buttons in the same view
- Use Accent Secondary buttons outside of marketing/CTA contexts
- Use Subtle buttons where context is unclear
- Create custom button styles outside these 4 types
- Use icon-only buttons without sufficient context
- Mix different button systems in the same interface

**Remember**: Always check these guidelines before implementing buttons to ensure you're using the right type for your use case.
